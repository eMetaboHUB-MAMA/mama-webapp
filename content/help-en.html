Users management Workflow

  • Service user
    any persone who wich gather intelligence and build project with MTH infrastructure node.
  • MetaboHUB staff
    any persone involved in MTH infrastructure and able to interact with clients in projects requests: she/he can be MTH Contact or MTH Project Manager ot MTH Analyst.
  • Administrator
    persone with unlimited access to the portal informations. She/He is also involved in users account management, portail checkup and maintenance.

Requests / projects management workflow

  • Waiting
    a new request has been submitted by a service user through the portal
  • Assigned
    The request has been processed by a MTH staff in charge to make an appointment with the service user in order to fullfil the request sheet together.
  • Accepted ou Refused
    routing point of the request after "Assigned", or of the project if it could not be funded (in this case its status should be "Rejected").
  • Completed
    the service user and a MTH staff have completed the request sheet. The request is now a "project" and is/ considered as processed by the platform(s) in charge. The MTH staff nomiate an "in charge" MTH staff and "involved" analysts on different platforms.
  • Running
    The project is being processed on the targeted platform.
  • Archived
    the project has been done - require a PF feedback and "finished" definition.
  • Paused
    request no ended or "accepted" but either waiting for funding or samples delivery.

  • Waiting request -> assigned request: when all fields have been fulfilled or are not "null".
  • Assigned request -> rejected / accepted request: the MTH staff in charge changes the request status after its evaluation.
